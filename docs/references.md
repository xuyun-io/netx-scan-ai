# Documentation References

This documentation set was restructured using common patterns from successful open-source AI projects.

## Reference Projects

| Project | Documentation pattern used as reference |
| --- | --- |
| [Dify](https://github.com/langgenius/dify) | Clear product positioning, Docker-first quick start, self-hosting guidance, and configuration examples. |
| [Open WebUI](https://github.com/open-webui/open-webui) | Self-hosted AI workspace positioning, feature-first README structure, and installation paths for different operators. |
| [LangGraph](https://github.com/langchain-ai/langgraph) | Clear agent-runtime positioning, architecture-oriented explanation, and separation between concepts and quick start. |

## Applied Principles

- Put the quickest successful path near the top of the root README.
- Keep deployment, API, configuration, and operations in separate pages.
- Avoid mixing historical research notes with user-facing documentation.
- Show copyable examples for common setup tasks.
- Document the current implementation honestly instead of describing aspirational features as complete.
- Keep secrets out of examples.
