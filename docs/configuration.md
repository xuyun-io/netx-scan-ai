# Configuration

NetX AI has two configuration layers:

- Application configuration in `agent-server/config/app.yaml`.
- AgentSpace configuration created through the UI or API.

Keep these layers separate. Application configuration controls the running Go service. AgentSpace configuration controls model access, runtime variables, and integrations for a specific agent workspace.

## Application Config

Create a local config file from the example:

```bash
cd agent-server
cp config/app.yaml.example config/app.yaml
```

The server always reads:

```text
agent-server/config/app.yaml
```

There is currently no alternate config path flag.

### Fields

| Field | Required | Default | Description |
| --- | --- | --- | --- |
| `httpAddr` | No | `:8080` | HTTP listen address. |
| `path.root` | No | `.` | Base directory used to resolve relative paths. |
| `path.agents` | No | `data/agents` | AgentSpace data directory. |
| `path.web` | No | `web/dist` | Built frontend static files. |
| `path.skills` | No | `skills` | Skill directory. |
| `publicURL` | No | empty | External browser URL used in task links, especially Enterprise WeChat messages. |
| `logLevel` | No | `info` | `debug`, `info`, `warn`, or `error`. |
| `logFormat` | No | `json` | `json` or `console`. |
| `auth.username` | No | empty | Basic auth username. |
| `auth.password` | No | empty | Basic auth password. |

### Local Example

```yaml
httpAddr: :8080
path:
  root: .
  agents: data/agents
  web: web/dist
  skills: skills
publicURL: http://localhost:5173
logLevel: info
logFormat: console

auth:
  username: ""
  password: ""
```

### Docker Example

```yaml
httpAddr: :8080
path:
  root: /app
  agents: data/agents
  web: web/dist
  skills: skills
publicURL: https://netx-agent.example.com
logLevel: info
logFormat: json

auth:
  username: admin
  password: replace-with-a-strong-password
```

## Public URL

`publicURL` is an application-level setting. It is not tied to an AgentSpace.

Use a URL that recipients can open from their browser:

```yaml
publicURL: https://netx-agent.example.com
```

Do not use a container-only address. Enterprise WeChat notifications build links like:

```text
{publicURL}/{agentSpaceName}/#/task/{taskId}
```

## Auth

When both `auth.username` and `auth.password` are non-empty:

- Static UI routes require browser login.
- All `/api/v1/*` routes require Basic auth except `/api/v1/login`.

When either value is empty, auth is disabled.

## Logging

Use JSON logs for containers:

```yaml
logFormat: json
logLevel: info
```

Use console logs for local development:

```yaml
logFormat: console
logLevel: debug
```

The backend logs service startup, HTTP requests, scheduler events, task lifecycle events, and notification status. It should not log API keys, webhook URLs, or full secrets.

## AgentSpace LLM Config

Configure LLM settings in each AgentSpace, not in `app.yaml`.

### Direct Gemini

```yaml
llm:
  provider: gemini
  model: gemini-2.5-pro
  apiKey: your-gemini-api-key
```

`google` and `google-ai` are accepted as aliases.

### Gemini Relay

Use this for a Gemini-compatible gateway, such as an internal relay or Tokenstars-compatible relay:

```yaml
llm:
  provider: gemini-relay
  model: gemini-2.5-pro
  apiKey: your-relay-api-key
  baseUrl: https://relay.example.com
```

Rules:

- `baseUrl` must be the relay root URL.
- Do not include `/v1beta`.
- The backend sends `Authorization: Bearer <apiKey>`.
- The Gemini native request path is generated by the SDK.

## AgentSpace Environment Variables

AgentSpace environment variables are passed to model and skill execution.

Common values:

| Variable | Purpose |
| --- | --- |
| `GOOGLE_API_KEY` | Gemini API key fallback for direct Gemini. |
| `GEMINI_API_KEY` | Gemini API key fallback for direct Gemini. |
| `GEMINI_RELAY_API_KEY` | Relay API key fallback for `gemini-relay`. |
| `TOKENSTARS_API_KEY` | Tokenstars-style relay key fallback. |
| `CC_SWITCH_API_KEY` | Internal relay key fallback. |
| Skill-specific endpoint variables | External endpoints required by enabled skills. |

The backend injects these runtime variables for skill execution:

| Variable | Purpose |
| --- | --- |
| `NETX_ARTIFACT_DIR` | Temporary directory where skills can write artifact candidates. |
| `NETX_TASK_ID` | Current task ID when a task is running. |

## Enterprise WeChat Integration

Enterprise WeChat is configured per AgentSpace:

```yaml
integrations:
  wecom:
    enabled: true
    webhookUrl: https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=...
```

Never commit real webhook URLs.
